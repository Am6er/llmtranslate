# LLM Translate — EN→RU (локальная модель)

Утилита (.NET 9 WinForms): переводит один файл (md/txt) на русский локальной моделью
через **OpenAI-совместимый эндпоинт** (`/v1/chat/completions`, по умолчанию
`http://127.0.0.1:8080`).

> ⚠️ **Сервер модели нужно поднимать самому — отдельно.** Приложение НЕ входит в комплект
> с моделью и НЕ запускает сервер: оно только подключается к уже работающему эндпоинту
> (см. раздел [«Свой сервер модели»](#свой-сервер-модели)).

## Что делает

- Мониторит `BaseUrl` из `config.json`, показывает **имя модели** и лампу (зелёная = связь есть),
  а рядом — **текущую температуру GPU**.
- Вход — **один файл** через диалог; выход — файл рядом с суффиксом **`_RU`** (`имя_RU.md`).
- Сохраняет формулы (`$…$`, `$$…$$`), код (```` ``` ````, `` `…` ``) и разметку markdown
  (заголовки, таблицы): защищённые фрагменты подменяются плейсхолдерами, переводится
  только проза (включая текст в ячейках таблиц), потом фрагменты возвращаются как были.
- Перевод идёт кусками **по целым абзацам** (абзац никогда не дробится); thinking у Qwen
  отключается (`chat_template_kwargs.enable_thinking=false`), генерация ограничена `MaxTokens`.
- Старт/стопа сервера тут **нет** — сервер поднимается отдельно (`full-server-9b-translate.cmd`).

## Устойчивость и возобновление

- **Чекпоинты (resume).** Рядом с исходником создаётся папка **`<имя>.progress`**: каждый
  переведённый чанк пишется в файл (`00000.part…`) + `meta.json` с хэшем документа. Если
  перевод оборвался (сеть/сервер) — повторный запуск **докатывает с места**, а не с нуля.
  По успешному завершению папка удаляется. Если документ или настройки изменились — чекпоинт
  считается устаревшим и перевод начинается заново.
- **Ретраи.** Каждый чанк повторяется при сетевых/серверных сбоях (`RetryCount`, backoff),
  прежде чем упасть.

## Термотроттл (бережём GPU)

Ползунок **«Держать GPU ≤ N°C»** (50–90°C). После каждого чанка температура GPU читается
через `nvidia-smi`, пишется в лог (`температура GPU: 61°C (цель ≤ 75°C)`), и если она выше
цели — приложение паузит по 3 c и перечитывает, пока карта не остынет до цели (потолок 120 c
на чанк, чтобы не зависнуть). Ползунок живой — двигается во время перевода, применяется со
следующего чанка. **90°C = без ограничений.** Прав администратора не требует.

## Конфиг (`config.json` рядом с exe)

| поле | назначение |
|------|-----------|
| `BaseUrl` | адрес:порт OpenAI-сервера (`http://127.0.0.1:8080`) |
| `Model` | id модели; пусто = авто из `/v1/models` |
| `SourceLang` / `TargetLang` | языки (по умолч. English → Russian) |
| `Temperature` | 0.3 (точный перевод) |
| `MaxCharsPerChunk` | размер куска перевода (по абзацам) |
| `MaxTokens` | потолок генерации на чанк (анти-runaway). Если больше контекста сервера (`n_ctx` из `/props`) — автоматически занижается до него с записью в лог |
| `RequestTimeoutSec` | таймаут запроса к модели |
| `OutputSuffix` | суффикс выходного файла (`_RU`) |
| `RetryCount` | число повторов чанка при сбое |
| `TargetTempC` | целевая макс. температура GPU (50–90; 90 = без ограничений) |
| `NvidiaSmiPath` | путь/команда `nvidia-smi` для чтения температуры |

Целевая температура правится прямо в окне (сразу пишется в `config.json`).

## Свой сервер модели

Приложению нужен любой **OpenAI-совместимый** сервер, отдающий `/v1/models` и
`/v1/chat/completions` по адресу из `BaseUrl`. Проще всего — `llama-server` из
[llama.cpp](https://github.com/ggml-org/llama.cpp) с GGUF-моделью (для EN→RU хорошо
подходит Qwen3.5-9B). Минимальный пример:

```bash
llama-server -hf unsloth/Qwen3.5-9B-GGUF:Q4_K_M --host 127.0.0.1 --port 8080 -c 8192
```

Подойдёт и любой другой OpenAI-совместимый бэкенд (LM Studio, vLLM, Ollama через `/v1`
и т.п.) — просто укажи его адрес в `BaseUrl`. Модель приложение выберет автоматически из
`/v1/models`.

> Thinking у Qwen приложение выключает само (`chat_template_kwargs.enable_thinking=false`),
> отдельно на сервере ничего настраивать не нужно.

## Готовая portable-сборка

GitHub Actions собирает **portable** сборку (self-contained, один `.exe`, .NET ставить не
нужно). Скачать: вкладка **Actions → последний build → артефакт
`LlmTranslate-portable-win-x64`**, а для тегов `v*` — из **Releases**. Распакуй и запусти
`LlmTranslate.exe` (рядом лежит `config.json`).

## Сборка из исходников

```powershell
dotnet build app/llmtranslate.csproj -c Release
# или portable-сборка одной командой:
dotnet publish app/llmtranslate.csproj -c Release -r win-x64 --self-contained true `
  -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -o publish
```

Порядок работы: подними свой сервер модели (см. выше) → дождись зелёной лампы с именем
модели → выбери файл → «Перевести → RU».

## Что подавать на вход

**RAG-markdown** (из llmocr), а не HTML: чистый markdown безопаснее для переводчика
(защита `$…$`/кода работает без помех, нет тегов, которые модель может поломать). HTML
генерируй уже из переведённого `.md`, если нужен просматриваемый вид.

## Ограничение

«Голые» переменные в тексте без `$…$` (одиночная буква) от обычных слов не отличаются и
могут быть переведены. Переменные в математических долларах защищены надёжно.
